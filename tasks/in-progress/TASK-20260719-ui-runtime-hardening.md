# TASK-20260719：工作台信息架构与 Bybit HY2 运行态收口

## 目标

在不删除既有功能的前提下，整理前端导航与运行态信息层级，并通过真实生产链路定位四条 Hysteria2 节点无法访问 Bybit 的具体边界。

## 范围

- 将导航按研究、验证、数据、系统职责重新分组，保留所有现有页面和 hash 入口。
- 将工作台失败记录、当前就绪度、常驻任务和快速操作分层，避免历史噪声覆盖当前状态。
- 对四条 HY2 节点逐条验证协议握手、HTTP CONNECT/SOCKS 转发、Bybit Demo API 与节点轮换。
- 生产公网、K8S 内部、浏览器和代理健康状态形成可复核证据。

## 非目标

- 不绕过 Bybit 风控、验证码或地区限制。
- 不关闭代理 fail-closed，不把 Bybit 代理静默改成直连。
- 不删除交易、研究、信息源、工作流和历史审计页面。

## 验收标准

- 桌面和移动端导航可扫描，所有原页面仍可达，主流程入口不超过两次点击。
- 工作台能区分当前阻断、历史失败和已完成运行；失败记录可直接跳转复核。
- 四条 HY2 节点分别有可重复的握手、代理和 Bybit 访问结果，失败原因可归类。
- CI、前端构建、生产浏览器 smoke、ArgoCD 与 Pod 状态全部通过。

## 测试方式

- Web Vitest、TypeScript/Vite build、生产公网 Playwright smoke。
- K8S proxy health、Bybit Demo 公网 API、节点级代理探测和后端日志核对。
