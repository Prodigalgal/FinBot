# TASK-20260722：前端 UI/UX 深度优化

## 目标

在不改变 API、数据契约和业务流程的前提下，将 FinBot 前端统一为高密度、易扫描、可响应式使用的研究机构交易终端，并解决当前视觉层级弱、组件细节不一致、移动导航低效和长表格难阅读的问题。

## 范围

- 重构全局 Theme、App Shell、桌面导航、移动抽屉和页面标题栏。
- 建立可复用的数据表面、状态标签、指标、区段标题和空/错/加载状态样式。
- 深化登录、研究发起、工作台、模拟交易与永久审计页面。
- 通过 MUI 全局组件覆盖统一其余页面的按钮、表单、表格、Tabs、Accordion、Alert 与 Paper。

## 非目标

- 不修改后端 API、数据库、状态机、鉴权流程和页面功能。
- 不删除现有导航、表单字段、配置项、交易事实或研究结果。
- 不引入新的前端框架、图表库、营销页面、虚假数据或代码生成。

## 文件 Ownership

- Codex：`src/ui.tsx`、Dashboard/Trading 集成、测试与最终验收。
- Grok：`src/theme.ts`、`src/App.tsx`。
- MiMo：`src/AuthGate.tsx`、`src/ResearchPage.tsx`。
- Reviewer 不得写入自己审查的分片；跨分片修改由 Codex 集成。

## 验收标准

1. 视觉风格与三张已接受概念图一致：真白/冷灰、石墨文字、钴蓝主色、语义状态色、无渐变、最大 6px 圆角。
2. 桌面导航、顶部栏、数据表格、状态标签和指标具有统一层级与交互状态。
3. 移动端使用导航抽屉，不再依赖占据首行的全宽 Select。
4. 390x844、1280x720、1536x1024 无横向溢出、遮挡、裁切和不可读文字。
5. 现有 13 页面导航、登录、研究发起、交易筛选和关键状态语义不退化。

## 测试方式

- `npm test`
- `npm run contract:check`
- `npm run build`
- Browser DOM/console/interaction 检查；截图能力不可用时使用 Playwright 截图。
- 概念图与最终桌面/移动截图通过 `view_image` 逐项对照。
