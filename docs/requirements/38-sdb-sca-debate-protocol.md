# SDB-SCA 辩论协议需求

## 目标

将现有按拓扑层轮流发言、由单一 Chair 终局裁决的辩论升级为
SDB-SCA（Simultaneous Double-Blind Game with Symmetric Social Choice
Architecture）。协议必须消除可见顺序造成的锚定效应、后发信息优势和单一裁判偏差，
同时保留可恢复、可审计和可解释能力。

## 范围

- 工作流可显式选择 `SDB_SCA_V1` 协议。
- 同一逻辑角色允许配置多个异构模型席位，但社会选择权重按逻辑角色归一。
- 辩论运行包含独立提案、双盲交叉评审、隔离修正、匿名投票和确定性聚合。
- 产生统一的 `CONSENSUS_RESULT`，供研究结论、自动化策略和模拟交易执行读取。
- 旧版 Chair 工作流只用于历史读取和在途恢复，不再作为新建主工作流的默认协议。

## 协议不变量

1. **同阶段隔离**：任何席位不得看到同阶段其他席位的未揭示产物。
2. **原子揭示**：阶段内达到 barrier 后一次性揭示全部有效产物，不能逐条泄漏。
3. **双盲评审**：评审输入只包含稳定匿名候选代号，不包含节点、角色、厂商、模型或消息身份。
4. **对称修正**：每个提案席位只收到针对自身匿名候选的完整评审集合，并同时修正。
5. **角色归一**：同一 `logicalRoleKey` 无论有多少席位，总投票权重均为 1。
6. **确定性终局**：使用 winning-votes Schulze 与确定性数值聚合，不允许 LLM 改写结果。
7. **失败关闭**：低法定人数、并列、顺序敏感、无有效选票或预测约束无效时输出 `UNCERTAIN`，禁止自动交易。

## 阶段

### 1. 同时独立提案

所有参与席位接收同一冻结研究快照并并行生成提案。产物先封存，只有所有必需任务完成、
超时或被判定不可恢复后才能跨越 barrier。

### 2. 双盲交叉评审

系统为候选生成确定性匿名代号。参与者独立评审其他候选；不超过 6 个候选时使用全矩阵，
超过 6 个候选时使用确定性均衡不完全分配，并保证每个候选满足配置的最小评审数。

### 3. 对称隔离修正

每个原始席位收到针对自身候选的所有已揭示匿名评审，同时独立生成修正版。修正阶段再次封存，
直至 barrier 原子揭示。

### 4. 社会选择

每个有效席位分别对同一匿名候选集合生成正序和逆序两份全排序或并列分层选票。
系统先在逻辑角色内部归一，再跨角色构造偏好矩阵并执行 winning-votes Schulze。
正序与逆序必须得到相同的唯一严格胜者，否则结果为 `ORDER_SENSITIVE`。

## 预测数值聚合

- 方向概率必须覆盖 `UP`、`SIDEWAYS`、`DOWN` 且总和满足容差。
- 先对同一逻辑角色内席位取均值，再对角色均值取均值。
- 目标价、止损和止盈先在逻辑角色内取中位数，再跨角色取中位数。
- 所有参考价格和预测区间来自提案阶段冻结的市场快照。
- 未来环境真值只用于预测窗口结束后的评估，不得追溯改变当时决策。

## 状态、恢复与审计

- 阶段、任务、产物、匿名映射、选票和决策必须永久落库。
- 任务 ID 必须由辩论、协议版本、阶段代际、阶段、席位和目标确定性生成。
- 同一任务 ID 重放相同内容必须幂等；不同内容哈希必须产生审计冲突并停止推进。
- barrier 推进使用数据库 CAS；Worker 重启后只能从已揭示阶段继续，不得读取半完成阶段内容。
- 运行态 API 必须能展示阶段、barrier 进度、匿名产物、偏好矩阵、排名、失败原因和恢复点。

## 工作流与兼容性

- 工作流版本持久化 `debateProtocol`，席位节点持久化不可变 `logicalRoleKey`。
- 新增 `SOCIAL_CHOICE` 节点类型和 `CONSENSUS_RESULT` 输出契约。
- SDB-SCA 工作流禁止席位之间通过内容边直接读取对方当前阶段输出。
- 旧版 `CHAIR_VERDICT` 仅用于历史数据和在途旧运行；下游优先读取 `CONSENSUS_RESULT`。

## 验收标准

1. 相同输入、相同有效产物在节点顺序和候选输入顺序置换后得到相同结果。
2. 三个同角色席位不能压过三个不同角色各一个席位的总权重。
3. 正反顺序选票胜者不一致时结果为 `UNCERTAIN`，且不产生自动交易指令。
4. Worker 在任一 barrier 前后重启均不会泄漏或重复生成不一致产物。
5. 同任务不同哈希、低 quorum、并列和畸形选票均有直接测试。
6. 前端能够区分“正在封存”“已揭示”“社会选择中”“无共识”等状态，并不展示模型真实身份。

## 非目标

- 不把 LLM 解释器作为终局裁判。
- 不使用 embedding 相似度决定角色克隆权重。
- 不在首版引入 Redis、MQ、分布式锁或多副本协调。
- 不要求密码学级秘密计算；单副本服务与数据库事务构成当前威胁模型下的隔离边界。
