# P2.2 人类可读的运营与 AI 决策体验

## 目标

- 将 Web UI 从工程调试台调整为面向研究与决策用户的运营工作台。
- 让用户进入系统后在一个视口内判断系统健康、最新结论、待处理事项和下一步动作。
- 将 AI 输出组织为“结论 -> 依据 -> 风险与下一步 -> 模型原文”，避免内部状态码和英文模型文本成为主要信息。
- 保留现有自动循环、研究流水线、交易建议、报告和配置能力，保持 advisory-only 边界。

## 范围

- `web-ui/src/App.tsx`
- `web-ui/src/OperationsPanels.tsx`
- `web-ui/src/humanReadable.ts`
- `web-ui/src/theme.ts`
- `tasks/current.md`

## 非目标

- 不修改自动循环、Council、风控门禁和产品筛选的后端业务规则。
- 不修改现有 API 或数据库契约。
- 不新增真实交易、下单、账户或私有交易所能力。
- 不用前端文案掩盖数据缺失、运行失败或研究未确认状态。

## 关键体验规则

- 总览优先显示运行状态、最终建议和注意事项，原始累计计数退到二级信息。
- 自动循环按“本轮结果、运行过程、多轮辩论、风险与治理”组织，默认先展示最终结果。
- 候选评分和门禁后置信度必须分开显示，并解释置信度归零或建议降级的真实原因。
- 多轮辩论按轮次分组；主席结论优先，角色原文按需展开。
- 交易建议和研究流水线必须同时展示提交控件、最近结果、后台状态、成功及失败反馈。
- 报告进入页面后自动载入，优先展示结构化摘要；原始 JSON 仅作为可展开的审计视图。
- 桌面端保留左侧导航；窄屏使用单一页面选择器，避免七个 Tab 横向滚动和发现成本。
- 长配置表单按站点折叠，保留保存、刷新、密钥脱敏和模型刷新能力。

## 影响文件

- `web-ui/src/App.tsx`：页面级导航、标题、状态和配置站点折叠。
- `web-ui/src/OperationsPanels.tsx`：总览、自动循环、交易建议、研究流水线和报告视图。
- `web-ui/src/humanReadable.ts`：状态、AI 文本、报告字段和运行步骤的人类可读映射。
- `web-ui/src/theme.ts`：工作台密度、表面层级和响应式控件细节。

## 验收标准

- 总览首屏能显示最近自动循环状态、下次运行、最终建议摘要和待处理事项。
- 自动循环不再同时展开全部 13 条辩论消息；用户可按三轮分析和主席结论切换。
- 最新三条建议能同时显示候选评分、门禁后置信度、研究状态、简要依据、主要风险和下一步。
- 英文 Chair/AI 决策原文不再作为默认主结论；原文仍可审计。
- 交易建议、研究流水线和报告页不再出现大面积无信息空白。
- 报告页不再默认只显示 JSON，切换报告类型后自动更新结构化摘要。
- 390x844 下无页面级横向滚动，自动循环首屏高度和操作路径明显收敛。
- 现有 AI 工作流编辑器、配置热更、模型刷新和后台任务能力不退化。

## 测试方式

```powershell
python -m compileall finbot
python -m unittest discover -s tests
cd web-ui
npm run build
```

- 浏览器验证：1536x1024 和 390x844。
- 页面覆盖：总览、自动循环四个视图、交易建议、研究流水线、报告、AI 站点、AI 工作流。
- 运行态验证：`/health`、`/api/v1/status`、`/api/v1/autonomous/status`，并确认 `FinBot-Web`、`FinBot-Worker` 常驻状态。
