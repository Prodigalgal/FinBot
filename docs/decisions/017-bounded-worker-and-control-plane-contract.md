# ADR-017：有界 Worker 与可生成的控制面契约

## 状态

Accepted，2026-07-16。

## 决策

1. 保留 Java 虚拟线程作为阻塞 I/O 执行载体，但在任务认领前同时检查全局和任务类型容量。数据库中的 `PENDING` 任务继续充当持久队列；进程内不建立无界等待队列。
2. lease 是外部副作用的 fencing 边界。heartbeat 续租失败后立即取消对应 `FutureTask`，执行线程不得再提交 `complete` 或 `fail`。
3. 行情序列采用 fail-closed：provider 返回的每一行都必须有效，时间戳严格递增且与请求周期连续。禁止通过静默丢弃或 `distinct` 修饰输入。
4. `contracts/finbot-control-plane.openapi.yaml` 是控制面固定路由的版本化契约。TypeScript 路径类型由该文件生成，CI 同时校验 Web 调用、公开 Controller 和契约三方覆盖。
5. 代理订阅中的不安全 TLS 标记不构成授权。默认过滤这些节点，只有显式运行配置才允许例外，并将例外暴露到健康状态。

## 取舍

- 使用数据库持久队列而非新增消息中间件，保持当前单副本部署简单，同时消除进程内无界并发。
- 任务取消依赖 handler 对线程中断和 `CompletionStage` 取消的响应；对不能被中断的远端调用，数据库 lease fencing 仍阻止本实例提交最终任务状态，但外部协议必须继续依靠幂等键恢复未知结果。
- OpenAPI 先约束路径和 HTTP 方法并生成前端路径类型，响应模型继续按业务优先级逐步从通用 JSON 收紧；CI 不再允许新增未登记路径。

## 回滚

- 并发上限可通过环境变量调高，但不得设为无界。
- 代理 TLS 例外可临时显式开启，健康状态会持续暴露风险。
- 契约检查可在紧急回滚中单独撤销，不影响运行时 API；不建议长期关闭。
