# ADR 030：清洗压缩阶段采用事实抽取语义

## 状态

已接受，2026-07-21。

## 决策

AI 清洗、压缩和独立验证阶段的目标是从原文抽取、去重并验证可追溯事实，不是描述文章讲了什么。

- 清洗席位剔除噪声和异常注入，并抽取实体、事件、时间、数值、条件、归因和不确定性。
- 压缩席位在事实级执行去重、同义合并和结构压缩，保留反例、单位、时效、适用条件和引用。
- 验证席位回看原文，拒绝元叙述、无来源断言、事实漂移、数值或单位错误、时间错位和错误归因。
- 现有 `CompressionContent.summary` 和数据库 `ai_compression.summary` 暂作为事实压缩正文的存储槽，不再表示文章摘要；`key_points` 表示原子事实集合。
- 发布不可变 `workflowversion_standard_v8`，保留 v7 供历史运行复现。

## 取舍

本次不迁移历史压缩产物，也不立即修改持久化字段名，避免把语义修正扩大成数据迁移。运行时 Prompt、v8 节点模板、节点名称和 operation 已统一为事实抽取语义；后续若需要结构化查询单条事实，再单独引入事实对象表与版本化契约。

## 验证

- 单元测试必须证明所有清洗、压缩和验证请求都包含原子事实要求、元叙述禁令和事实压缩正文约束。
- PostgreSQL Liquibase 集成测试必须证明 v8 是唯一 published 版本，五个证据节点均使用事实抽取 Prompt，且拓扑和 Provider 路由未退化。
