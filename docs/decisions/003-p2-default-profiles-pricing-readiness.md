# ADR-003：采用显式配置档案与型号绑定费率

日期：2026-07-11

## 决策

P2 使用代码内置、版本化的配置档案提供开箱默认值，但不在进程启动时自动写盘。用户通过 setup API 或 Web 明确应用后，系统才把非敏感值写入 `runtime_config.json`。

AI 费率绑定到明确模型。站点级输入/输出字段继续作为执行时成本接口，但只有调用模型等于 `pricing_model` 时才视为 known。默认输入费率采用 cache miss 价格，使预算门禁保持保守。

治理报告可以对历史 invocation 使用当前匹配费率重算成本。重算只存在于新治理报告中，原始 invocation 保持不可变，并在 summary 中记录 `repriced_invocation_count`。

## 取舍

- 显式应用避免首次启动就产生网络请求或 AI 费用。
- fill-missing 默认语义减少配置工作，同时保护用户已有调整。
- 提供显式 overwrite 能让用户切换档案，但仍排除所有敏感字段和代理策略。
- 型号绑定避免用户刷新模型后继续套用旧价格。
- 内置价格会随厂商调整而过期，因此必须展示来源和核对日期，允许用户覆盖。

## 兼容与回滚

- AI 配置版本递增；旧配置缺少费率元数据时，对已知内置站点补默认值。
- 自定义站点保持费率 unknown，除非用户显式填写。
- Runtime 配置格式不变，档案只是现有 `RuntimeConfigStore.update()` 的受控调用方。
- 回滚时可停止使用 setup API；已写入的非敏感运行时值仍可从现有配置界面修改或清除。
